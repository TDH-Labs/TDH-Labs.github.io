Why Harbor

The agent environment,
not the agent.

Every AI coding tool wants to own your workflow. Harbor inverts that — it's the environment your agents run inside, not another agent you have to learn. Bring your own models, your own tools, your own way of working.

🧩

Agent-Neutral

Works with Claude Code, Codex, OpenCode, Cursor, goose, or any MCP-compatible agent. Harbor doesn't care which agent you use — it provides the runtime, context, and tool wiring underneath.

🔒

Self-Hosted

Everything runs on your machine or your infra. No telemetry, no cloud dependency, no vendor gatekeeping. Your prompts, context, and code never leave your network unless you route them out. Harbor is a cooperative control plane, not an OS sandbox — agents share your user permissions and filesystem.

🧠

Persistent Context

Agents lose context between sessions. Harbor doesn't. Structured memory, project state, and cross-session continuity that survives model swaps, agent changes, and terminal restarts.

📊

Budget Enforcement

Every session carries a token budget, checked and debited in-process on each skill load. Spending is tracked in real time and visible on the local dashboard — no agent quietly blows past its limit.

📁

Workspace Management

Each project gets a separate workspace with its own tools, models, and context store. Switch between projects without cross-contamination of context or config. Workspaces are organizationally separated by cooperative convention, not OS-level isolation — agents in different workspaces share your user account.

⚙️

Pluggable Tool System

Tools are just MCP servers or local scripts. Harbor discovers and wires them automatically. File system, web search, code execution, databases — add any tool your agents need without framework gymnastics.

Quick Start

Set up Harbor bash
# Requires Bun ≥ 1.1 · alpha — expect breakage # Install from the alpha tarball (npm release lands at beta) npm i -g ./harbor-tugboat-0.1.0.tgz # Stand up the environment harbor init # seed agent_map.md + AI beacons harbor setup # build the directory tree from config harbor check # verify everything is wired # Route an agent's skill access through Harbor harbor install --for claude-code
Config: config.toml toml
# config.toml — rooms, skills, budgets [paths] home = "~" state_dir = "~/.agent-env" # A room scopes which skills + MCP servers a session reaches [skills.rooms.research] skills = ["web-search", "summarize"] capabilities = ["read_skill", "list_skills"] [[skills.rooms.research.mcp.servers]] name = "filesystem" command = "npx" [budgets] default_session_limit = 100000
Trust model

A control plane, not a sandbox.

Harbor governs how cooperating agents load skills, spend budget, and enter rooms — and records every decision. It is not a cage for a hostile process. Here is exactly where the line sits.

🛡️

What Harbor enforces

✓
Gated skill access
Every skill load runs a room + budget + audit check.
✓
In-process budgets
Token limits debit on the hot path — no quiet overspend.
✓
Full audit trail
Every allow and deny logged with room, session, reason.
⚠️

Where the boundary ends

—
Not an OS sandbox
Raw file access can read a skill directly, skipping the gate.
—
Rooms are cooperative
The room comes from AGENT_ENV_ROOM; Harbor trusts it.
—
Open by default
A room with no skill list allows all; gating is opt-in.
Harbor makes the cooperative path the easy, observable, budgeted one. It does not claim to be unbypassable OS-level isolation — that is a separate layer, on the roadmap, not in 0.1.
Get Started

From zero to running in minutes

No accounts to create. No SaaS to sign up for. Just install, configure, and go.

01

Install

Alpha ships as a tarball — npm i -g ./harbor-tugboat-0.1.0.tgz. Needs Bun ≥ 1.1. No Docker, no cloud, no account.

02

Configure

One TOML file — config.toml — defines your rooms, the skills and MCP servers each can reach, and per-session budgets.

03

Connect an Agent

Harbor exposes an MCP server and a CLI. Connect Claude Code, Codex, Cursor, goose, or any MCP-native agent. No adapter needed.

04

Ship

Your agent gets persistent context, gated tool access, in-process budgets, and a full audit trail — out of the box. Focus on building.
