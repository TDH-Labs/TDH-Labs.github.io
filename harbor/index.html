Harbor

A local agent-neutral control plane

Self-hosted AI agent environment that manages context across models, tools, and workspaces — without vendor lock-in. Bring your own LLMs. Keep everything local. Ship with any agent framework.

GitHub → Learn More ↓
Why Harbor

The agent environment,
not the agent.

Every AI coding tool wants to own your workflow. Harbor inverts that — it's the environment your agents run inside, not another agent you have to learn. Bring your own models, your own tools, your own way of working.

🧩

Agent-Neutral

Works with Claude Code, Codex, OpenCode, Hermes, goose, or any MCP-compatible agent. Harbor doesn't care which agent you use — it provides the runtime, context, and tool wiring underneath.

🔒

Self-Hosted & Private

Everything runs on your machine or your infra. No telemetry, no cloud dependency, no vendor gatekeeping. Your prompts, context, and code never leave your network unless you route them out.

🧠

Persistent Context

Agents lose context between sessions. Harbor doesn't. Structured memory, project state, and cross-session continuity that survives model swaps, agent changes, and terminal restarts.

🔌

Multi-Model Routing

Route different tasks to different models. Cheap model for file search, capable model for architecture, fast model for edits. Run local GGUF quantized models alongside cloud APIs from a single config.

📁

Workspace Management

Each project gets an isolated workspace with its own tools, models, and context store. Switch between projects instantly without cross-contamination. Per-project skill libraries and memory stores.

⚙️

Pluggable Tool System

Tools are just MCP servers or local scripts. Harbor discovers and wires them automatically. File system, web search, code execution, databases — add any tool your agents need without framework gymnastics.

Quick Start

Install Harbor bash
# Install the CLI pip install harbor-agent # Initialize a workspace harbor init my-project cd my-project # Configure your models harbor config set model.default anthropic/claude-sonnet-4 harbor config set model.fast local/llama-3.2-3b # Run any agent in Harbor harbor run "Build a REST API for task management"
Config: harbor.yaml yaml
# harbor.yaml — single source of truth models: default: "openai/gpt-5" fast: "ollama/llama3.2:3b" memory: backend: "sqlite" path: "~/.harbor/memory.db" tools: filesystem: true web_search: true code_execution: true
Get Started

From zero to running in minutes

No accounts to create. No SaaS to sign up for. Just install, configure, and go.

01

Install

pip install harbor-agent or grab the binary from GitHub releases. No dependencies, no Docker, no cloud.

02

Configure

Point Harbor at your preferred LLMs — local, cloud, or both. One YAML file controls models, tools, memory, and workspaces.

03

Connect an Agent

Harbor exposes an MCP server and a CLI. Connect Claude Code, Codex, Hermes, goose, or any MCP-native agent. No adapter needed.

04

Ship

Your agent gets persistent context, tool access, multi-model routing, and workspace isolation — out of the box. Focus on building.
